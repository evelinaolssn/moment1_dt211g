Om uppgiften

Planering och förberedelse

För att skapa en tydlig struktur började jag med att skapa de mappar som skulle användas för projektet, bland annat en src/mapp där arbetfilerna skulle lagras och en dist/mapp där de minifierade versionerna skulle lagras med hjälp av verktyget Parcel. Jag installerade parcel med hjälp av kommandot "npm install --save-dev parcel" i terminalen då paketet endast skulle användas för att bygga och optimera projektet. Därefter skapades en .gitignore fil för att säkerställa att onödiga kataloger inte ska versionshanteras. Grundläggande kod implementerades i HTML-, CSS- och JavaScript filerna innan första commit gjordes till main branchen i GitHub-repositoryt för projektet.

Branches

Efter att projektet hade fått sin grundläggande struktur och första commit genomfördes publicerade jag webbplatsen genom att koppla projektets main-branch till Netlify via GitHub. För att fortsätta utvecklingen och göra ändringar i projektet skapade jag en separat developer-branch, där alla commits under utvecklingsprocessen genomfördes. Syftet var att senare slå ihop ändringarna med main-branch när utvecklingen var klar.

Uppdatering av kod och optimerade bilder med hjälp av Parcel

I detta steg uppdaterade jag HTML strukturen för att anpassa det till projektet samt lade till bilder i mappen images som tidigare var tom. Med hjälp av verktyget Parcel konverterade jag bilderna till WebP-format samt minskade bildernas bredd genom att använda "srcset". Därefter genomfördes en commit till developer-branchen.

Browserlist-konfiguration

I detta steg uppdaterade jag package.json-filen för att säkerhetsställa att Parcel optimerar koden korrekt. Detta gjordes genom att lägga till browserlist-konfiguration för att specifiera vilka webbläsare och versioner som webbplatsen ska stödja.

Utveckling

Utöver detta genomfördes commits till developer-branchen när css- och Javascript-kod implementerades, som media queries och hamburgarmeny för responsiv design. Undersidorna uppdaterades med information om uppgiften och tillvägagångssätt, där slutligen arbetet slogs ihop med main-branchen.

Sammanfattning

Sammanfattningsvis har verktyget Parcel använts för att automatisera och effektivisera utvecklingen, samt för att minifiera både JavaScript, CSS och bilder. För att skapa ett organiserat arbete har jag använt mig av två separata branches, developer och main, där main-branchen varit direkt kopplat till Netlify där webbplatsen publicerats. Under utvecklingen har jag använt mig av kommandot "npm run start" för att säkerställa att funktionaliteten fungerade som förväntat. När projektet var klart användes kommandot "npm run build" för att publicera en färdig version av webbplatsen.
